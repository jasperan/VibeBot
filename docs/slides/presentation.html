VibeBot

Discord voice conversation + music bot

Voice AI Music Discord Python

Why VibeBot?

Discord voice channels are for hanging out. But bots can't hold a conversation.

  • Text-only bots force you to type while talking
  • Music bots play audio but can't interact
  • Voice assistants exist, but not inside Discord
  • No single bot does both: talk and play music

Two Modes, One Bot

🎙️

Voice Mode

Talk in a voice channel and the bot responds conversationally. Real-time speech-to-text, LLM generation, text-to-speech.

/listen
🎵

Music Mode

Play YouTube audio with standard queue controls. Search by name or paste a URL.

/play

Audio Processing

  • Input: Discord sends 48kHz stereo PCM
  • Downmix: Stereo to mono averaging
  • Resample: 48kHz to 16kHz for ASR
  • Output: TTS at 24kHz, upsampled to 48kHz stereo
def resample_pcm(pcm_data, from_rate, to_rate):
  samples = np.frombuffer(pcm_data, np.int16)
  ratio = to_rate / from_rate
  new_length = int(len(samples) * ratio)
  indices = np.linspace(0, len(samples)-1,
    new_length).astype(int)
  return samples[indices].tobytes()

Voice Activity Detection

WebRTC VAD segments speech from silence in real-time.

30ms Frames

Audio is chunked into 30ms frames (960 bytes at 16kHz). Each frame gets a speech/silence verdict.

Silence Threshold

Configurable silence duration triggers end-of-utterance. Prevents cutting off mid-sentence.

Max Utterance Cap

Long monologues get chunked at a configurable max length. Keeps the pipeline responsive.

ASR Client

Converts speech to text using Microsoft VibeVoice, served through vLLM.

  • PCM audio wrapped in WAV format
  • Base64-encoded as a data URL
  • Sent via OpenAI-compatible chat API
  • Handles both JSON and plain text responses

  • Sliding context window (configurable depth)

TTS Client

VibeVoice Realtime streams audio chunks over WebSocket. No waiting for the full synthesis.

  • WebSocket connection per synthesis call
  • 24kHz PCM audio chunks streamed back
  • Configurable voice, CFG scale, and diffusion steps
  • Default voice: "Carter"

Music System

YouTube search and playback powered by yt-dlp and FFmpeg.

  • yt-dlp extracts best audio stream URL
  • FFmpeg decodes and streams to Discord
  • Volume control via PCMVolumeTransformer
  • Auto-advances queue when tracks end

Design Decisions

Async Everything

httpx.AsyncClient and websockets keep the event loop free. No blocking calls.

Sliding Context

Conversation context window keeps the last N turns. Old messages get evicted to control token usage.

PCMStreamSource

Custom Discord AudioSource that buffers TTS chunks and serves 20ms frames on demand.

Mode Mutex

Voice and music can't run simultaneously. Cogs check each other's state before starting.

Testing

Full test suite covering every component, from audio utilities to cog interactions.

  • Unit tests for each client (ASR, LLM, TTS)
